Return Recovery Dashboard

User Journey — Returns Receiving SLA Breach

End-to-end flow from alert discovery to escalation and loop closure. Persona: Supply Chain Head at HQ.

Phase 1 — Entry
User Action
Step 01
Supply Chain Head opens the Return Recovery Dashboard
Arrives at HQ and opens the dashboard on desktop to monitor warehouse operations and close active leakages — Return Restocking Delay, Return Receiving Delay, Wrong QC, and others.
Phase 2 — Alert Discovery
User Action
Step 02
Spots the SLA Breach alert at the top of the Copilot list
The Copilot sidebar surfaces a critical alert at the top of the queue.
🔴
Alert title: Returns Receiving SLA Breach · ₹4.57L  SLA Breached
📋
Subtext: 234 return shipments across 10 FCs have breached their receiving SLAs
Phase 3 — Investigation
User Action
Step 03
Clicks the alert — flyout opens with full detail
Curious why ₹4.57L is at risk, the user clicks the card. The flyout slides in with three structured sections to read through.
⚠️
What’s at stake: ₹4.57L untracked returns exposed to pilferage · SPF claim timeline running since courier delivery date
⌛
What triggered this: Reverse pickups and RTOs found where no manifest or putaway was created within the receiving SLA for respective facilities
📦
Breakdown table: Toggle between Facility view and Courier view — breached count, value, oldest breach date, and inline CTAs per row
